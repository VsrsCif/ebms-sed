Izgradnja

Projekt ebms-sed je sestavljen s pomočjo maven 3.x tehnologije. Za izgradnjo aplikacije je potrebno imeti nameščeno java 1.8+ in maven 3.x.
Aplikacija uporablja Java Cryptography Extension (JCE). Za pravilno izgradnjo in delovanje je potrebno namestiti dodatek z varnostnimi pravili "Java Cryptography Extension (JCE) Unlimited Strength" v mapo: "/lib/security".
Dodatek je za javo 1.8 dosegljiv na naslovu: http://www.oracle.com/technetwork/java/javase/downloads/jce8-download-2133166.html.

Na spletnem naslovu: https://github.com/VsrsCif/ebms-sed prenesi izvorno kodo lokalno na disk. (Priporočamo z klikom na Gumb DownloadZip. Preneseni arhiv ebms-sed-master.zip odpakiraj v mapo "ebms-sed-master".)
V ukazni vrstici (linux: bash, windows: cmd), se postavi v odpakirano mapo in poženi maven ukaz za izgradjo aplikacije: 
mvn clean install

Po kočnani izgradnji so narejeni naslednji moduli:
  • ebms-sed-dao.jar: implementacija skupnih storitev za shranjevanje in branje entitet.
    Modul se nahaja v podmapi:
    ebms-sed-dao/target/ebms-sed-dao.jar
  • ebms-msh.ear: implememntacija spletnih ebMS 3.0 storitev za izmenjavo ebMS 3.0 sporočil med MSH moduli. Sporočila so podpisana/kriptirana skladno z p-mode konfiguracijo.
    Po uspešni namestivti na lokalni računalnik, so storitve dostopne na naslovu: http://localhost:8080/ebms-sed/ebms?wsdl.
    Modul se nahaja v podmapi:
    ebms-msh-module/ebms-msh-ear/target/ebms-msh.ear
  • ebms-sed-ws.war: implementacija spletnih storitev za integracijo z zalednimi sistemi. Po uspešni namestitvi so storitve dostopne na naslovu: http://localhost:8080/ebms-sed-ws/sed-mailbox?wsdl.
    Modul se nahaja v podmapi:
    ebms-sed-module/ebms-sed-ws/target/ebms-sed-ws.war
  • ebms-sed-webgui.war: implementacija spletnega vmesnika za nadzor delovanja aplikacije. Po uspešni namestitvi je aplikacija dostopna na naslovu: http://localhost:8080/ebms-sed-webgui/.
    Modul se nahaja v podmapi:
    ebms-sed-module/ebms-sed-web/target/ebms-sed-webgui.war
  • plugin-zpp.war.war: implementacija ZPP plugin modula, ki je implementacija SVEV 2.0 koreografijo izmenjave sporočil. Po uspešni namestitvi so administracijske strani dostopne: http://localhost:8080/zpp-plugin/.
    Modul se nahaja v podmapi:
    ebms-sed-module/ebms-sed-web/target/ebms-sed-webgui.war
Po kočnani izgradnji so narejeni naslednje skupne knjižnice:
  • ebms-msh-xsd-1.0.jar: v knjižnici so generirani java objekti izgrajenih iz shem: SOAP_1.1.xsd (http://schemas.xmlsoap.org/soap/envelope/) ,SOAP_1.2.xsd (http://www.w3.org/2003/05/soap-envelope/), xml.xsd (http://www.w3.org/2001/03/xml.xsd) , ebms-header-3_0-200704.xsd (http://docs.oasis-open.org/ebxml-msg/ebms/v3.0/core/ebms-header-3_0-200704.xsd) , ebbp-signals-2.0.xsd (http://docs.oasis-open.org/ebxml-bp/2.0.4/ebbp-signals-2.0.4.xsd) , xenc-schema.xsd (http://www.w3.org/TR/xmlenc-core/xenc-schema.xsd) , xmldsig-core-schema.xsd (http://www.w3.org/TR/xmldsig-core/xmldsig-core-schema.xsd) , xlink.xsd (http://www.w3.org/TR/xml-i18n-bp/xmlspec/xlink.xsd);
    knjižnica se nahaja v podmapi:
    ebms-sed-libs/ebms-msh-xsd/target/ebms-msh-xsd-1.0.jar
  • ebms-sed-wsdl-1.0.jar: v knjižnici je izgranjen java klient (API) spletnih storitev modula ebms-sed-ws. Knjižnica se lahko uporabi za izdelavo integracije zalednih sitemov preko spletnih storitev z ebms-sed aplikacijo ;
    knjižnica se nahaja v podmapi:
    ebms-sed-libs/ebms-sed-wsdl/target/ebms-sed-wsdl-1.0.jar
  • ebms-sed-commons-1.0.jar: Knjižnica združuje skupna orodja in statične šifrante;
    knjižnica se nahaja v podmapi:
    ebms-sed-libs/ebms-sed-commons-wsdl/target/ebms-sed-commons-1.0.jar

Namestitev aplikacije

v mapo wildfly-10.0.0.Final/standalone/ 

copy "[buildFolder]\ebms-sed\ebms-msh-module\ebms-msh-ear\target\ebms-msh.ear" [wildfly-home]\standalone\deployments\
copy "[buildFolder]\ebms-sed\ebms-sed-module\ebms-sed-ws\target\ebms-sed-ws.war" [wildfly-home]\standalone\deployments\
copy "[buildFolder]\ebms-sed\ebms-sed-module\ebms-sed-web\target\ebms-sed-webgui.war" [wildfly-home]\standalone\deployments\

Priprava sed-home mape

V mapi [buildFolder]\ebms-sed\ebms-sed-standalone\home_dir_template je primer prednastavljenih datotek. Presnemi mapo v c:\temp\home-dir) 
copy  [buildFolder]\ebms-sed\ebms-sed-standalone\home_dir_template c:\temp\sed-homer
Nato ponovno poženi strežnik z naslednjimi nastavitvami: 
./standalone.sh -c standalone-ebms.xml -Dorg.sed.msh.hibernate.hbm2ddl.auto=create -Dsed.home=c:\temp\sed-home
OPOZORILO
aplikacijo s parametrom org.sed.msh.hibernate.hbm2ddl.auto=create poženi samo prvič. Parameter omogoči izdelavo baznih tabel.
